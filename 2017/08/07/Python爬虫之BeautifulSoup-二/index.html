Python爬虫之BeautifulSoup(二)

发表于 2017-08-07 | | 阅读次数

上篇文章主要讲了关于Beautiful Soup的一些基本用法，这里我们继续聊下关于它的一些详细用法。在实际的使用中，其实我们用的最多的一个对象可能就是Tag了，今天主要也是讲通过使用Tag的各种属性方法来获取我们需要的内容。

1.遍历文档树

我们使用HTML内容，构造Beautiful Soup对象

（1）子节点

注意这里的子节点，都是指Tag的子节点，字符串节点是没有子节点的。

  • contents

其中tag的.content属性，可以将tag的子节点以列表的形式输出
例如：

输出：

[<li>data</li>]

既然获取到的是一个列表，那我们就可以用索引的方式去获取想要的内容了

输出：

<li>data</li>

  • children

    tag的.children返回的是一个list生成器对象
    我们打印下看看输出内容：

输出：

<list_iterator object at 0x106a10550>

既然children返回的是list生成器，那我们可以通过
可以通过for循环遍历来获取内容：

输出：

<list_iterator object at 0x10a21ae80>
<p class="title" name="name1"><li>data</li></p>
<p class="title2" name="name2">data</p>
<a class="aClass" href="http://www.baidu.com" id="link"><!-- 注释内容：Baidu --></a>

（2）子孙节点

tag的content和children属性仅包含直接tag的子节点,descendants属性可以对所有的子孙节点进行递归循环，它返回的同样是一个list生成器

for desChild in soup.p.descendants:
    print(desChild)

输出：

<li>data</li> data

从输出结果我们可以看到，所有的子节点内容会被一层一层的剥离出来。

（3）节点内容

如果tag只有一个 NavigableString 类型子节点,那么这个tag可以使用 .string 得到子节点:

输出：

This is title

如果一个tag仅有一个子节点,那么这个tag也可以使用 .string 方法,输出结果与当前唯一子节点的 .string 结果相同:

输出：

data

（4）多个内容

.string

beautiful soup提供了获取多个内容的属性方法.strings,获取的是一个list生成器，我们可以通过遍历去获取内容

输出：
This is title
data
data

注意：这里遍历获取到的内容并不包含注释

.stripped_strings

通过.strings获取到的内容可能会包含很多空格或换行符，使用.stripped_strings可以去除这些无用的空白字符

（5）父节点

通过.parent属性，我们可以获取到任意tag或者字符串的父节点

输出：

<head><title>This is title</title></head>

输出：

<p class="title" name="name1"><li>data</li></p>

输出结果我们可以看到，.parent可以获得父节点的全部内容

通过.parents可以获取所有父节点的内容

输出：

head html [document]
